2. Project Objective

Students will build a telecom operations application where:

  • Customers can register from portal
  • Customers can raise telecom complaints
  • Complaint agents can manage complaints
  • Incidents will be automatically created
  • Network outages can be tracked
  • Field engineers can receive tasks
  • SLA can monitor resolution time
  • Notifications and events will trigger automatically
  • Reports and dashboards will monitor KPIs

3. Concepts Covered

This project helps students practice a wide variety of ServiceNow development concepts in a real-world scenario.

PHASE 6 — SERVER SIDE SCRIPTING

UI Policies & Data Policies

UI POLICY 1 — CRITICAL COMPLAINT

  • Table: Complaint Table
  • Condition: Impact Level = Critical
  • Actions: Show Escalation Comments, Make Escalation Comments mandatory.

UI POLICY 2 — OUTAGE REQUIRED

  • Table: Complaint Table
  • Condition: Complaint Type = No Network
  • Action: Make Related Outage mandatory.

DATA POLICY 1 — MOBILE MANDATORY

  • Table: Customer Table
  • Field: Mobile Number
  • Behavior: Mandatory = True

DATA POLICY 2 — DESCRIPTION REQUIRED

  • Table: Complaint Table
  • Field: Description
  • Behavior: Mandatory = True

Business Rules

Outage Table | After Insert

Purpose: Automatically create engineer task.

BR 6 — CLOSE OUTAGE

Eng Task Table | After Update

Final Beginner Rules

RULE 1 Use existing Task fields. Do not create duplicate fields.
RULE 2 Keep Business Rules simple.
RULE 3 Keep Flow Designer simple. Maximum 3–5 actions.
RULE 4 Use Record Producers in Service Portal.
RULE 5 Avoid advanced scripting.
RULE 6 Test every module separately.
RULE 7 Keep forms simple and clean.
RULE 8 - Always test before final demo: SLA, Notifications, Assignments, Portal submission, Incident creation.

Final Project Outcome

After completing this project students will understand CSA fundamentals, Incident management, Task management, Table relationships, Service Portal, Flow Designer, Notifications, SLA, ACL, Client Scripts, UI Policies, Business Rules, Script Includes, Reporting, Dashboard creation, and the End-to-end ServiceNow development lifecycle.
